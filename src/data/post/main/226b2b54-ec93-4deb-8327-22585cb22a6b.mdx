# 새로운 JAR에 대한 자동 실행 프로그램 (UEC)

### 자동 실행 구축

백엔드 서버에서 새로운 jar이 들어올 경우 이를 감지하고 자동으로 실행이 되게 만들고자 했습니다.

이로써 후보 시스템이 세가지가 있었습니다.

1. infotify-tools

기존의 SSH 파일 전송을 유지하면서,
리눅스 커널에서 제어가 가능했지만 문제점이 존재했습니다.

SSH 파일 전송은 기존 파일이 존재할 경우에 이를 덮어씌우게 되었고 해당 툴은 그것에 대한 변경을 무조건적으로 감지해버리고, 실행을 한다는것이였죠.

이렇게 되면은 에러가 발생했을 경우에 새 파일이 들어오기 이전에 파일로 되돌아가지 못하고 계속 에러가 발생한 파일을 보유하게 된다는 문제가 있었습니다.

2. PHP

언어 특성상 빠르고, 다중 제어에 탁월하지만 아직 해당 언어를 능숙하게 사용하지 못해 후보군에서 제외했습니다.

3. Python + Flask

위와 같은 이유로 해당 프레임워크를 사용하기로 했습니다.

사용한 주요 라이브러리는 다음과 같습니다.

1. [watchdog](https://pypi.org/project/watchdog/)

파일의 입출력 이벤트를 감지하는 라이브러리입니다.

2. [psutil](https://pypi.org/project/psutil/)

프로세스를 다루는 라이브러리 입니다.

기본 파이썬 라이브러리에서는 각 환경에 따라서 다르게 제어를 했어야하지만 해당 라이브러리는 그런 요소들을 획기적으로 줄여줍니다.

### 주요기능

* 파일 업로드
  * 서비스 자동 등록
* 파일 변경 감지
* 버전 관리
* 실행 및 롤백

자세한 사항은 [제어 코드](https://github.com/terrinens/UploadEvent) 소스를 참조하십시오.

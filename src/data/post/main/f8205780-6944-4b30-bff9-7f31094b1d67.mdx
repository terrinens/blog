# Git 조직 비공개 레파지토리에 대한 요청 방식

해당 Git의 조직 레파지토리가 기본적으로 Privaet로 설정되어있어서 무조건 권한이 필요했다.

그래서 내 계정은 정상적으로 조직 레파지토리에 접근이 가능하니 내가 토큰을 만들어 권한을 할당하면 가능하겠구나라고 생각했다.

그렇기에 아래와 같이 권한을 할당했다. 보안성을 위해 극히 일부 권한만 주어져야했다.

결과는 성공적이였다! (이후 /release 도 요청 해보았다)

내가 만들었던 토큰으로 요청을 했더니 성공적으로 반환이 되었다.

작업을 하던 도중에 무언가 이상한점을 발견했다.

git의 주소는 개인과 조직의 주소 자체값이 달랐던거 같은데…?

하지만 여전히 요청하는 방식은 동일하다는것을 발견했다.

하지만 의심해볼만한 부분이라는건 분명했다.

그래서 [git api 문서](https://docs.github.com/ko/rest/repos/repos?apiVersion=2022-11-28#list-organization-repositories)를 참조했다.

그리고 해결책을 발견했다….

기존 : [https://api.github.com/repos/OWNER/REPO/](https://api.github.com/repos/OWNER/REPO/releases)

조직 :  [https://api.github.com/orgs/ORG/repos](https://api.github.com/orgs/ORG/repos)

요청 해야하는 주소가 달랐던것이다!

그래서 당장 시도해봤다.

결과는 Forbidden…. 권한 부족이였다.

권한이 오너로 승격되고 나서 다시 한번 시도 했다.

레파지토리가 정상적으로 조회되기 시작했으나, private 레파지토리는 조회가 되지 않는 현상이 발생하기 시작했다.

여러가지 방법을 사용해보기도 요청 인수에 invisilvle = private를 요청해봐도 여전히 빈값이였다.

그래서 한번더 토큰에 권한을 보기로 했을때 떠오른것이 있었다.

어떤 액세스에 대한 토큰인지 설정하는 부분이 존재했다는거!

그래서 조직 설정 페이지로 들어가 액세스 토큰을 허용 하는 페이지를 찾았다…

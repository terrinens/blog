# 헤더 값이 넘어오지 않던 문제

해당 문제점을 해결 하면서 얻어낸 이미지 입니다. 한번 가보고 싶네요.

**경상북도 경주시 충효동 670 옥상**

## 자세한 문제사항

### 😭 무슨 문제 사항이 있었나요?

리액트에서 로그인 요청을 시도 했을 경우에 커스텀 헤더 값을 가져오지 못하는 문제가 발생했습니다.

### 🧐 어디가 문제였었나요?

백엔드 cors 설정에서 헤더 응답값을 사용 할 수 있겠끔 설정을 추가해줬어야 했습니다.

## 해결 방안

WebMvcConfigurer 의 addCorsMappings 에서
exposedHeaders에 설정값을 사용해 커스텀 헤더를 사용 할 수 있게 개방을 했습니다.

이와 같은 설정은 [해당 문서](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Access-Control-Expose-Headers#syntax)에서 자세히 알아볼수 있었습니다.

```txt
Access-Control-Expose-Headers: [<header-name>[, <header-name>]*]
Access-Control-Expose-Headers: *
```

## 😡 인텔리제이 .http 는 왜 가능했을까?

하지만 의문점이 하나 있었습니다.

똑같은 외부 URL 요청을 했을 경우에는 리액트에서는 불가능했지만 .http에서는 허용하지 않은 헤더값도 잘 가져 올 수 있었습니다.

### 소스코드 뜯어보기

이것이 어떻게 가능했는지 한번 소스 코드를 뜯어보았습니다.

하지만 소스 코드를 뜯어봐도 아무것도 얻지 못했고, 얻어낸건 .http가 타입스크립트를 사용하고 있다는거였습니다.

또한 .http 가 어떻게 작동하는지 알아보기 위해서 여러 방면으로 찾아봤지만 해독기가 무엇인지 알아낼 방법이 없었으므로 다른 방면을 찾아봤어야 했습니다.

일단은 여러가지 **우회 방법을** 찾았지만 대부분의 우회 방법이라고 소개되는것들은 백엔드에서 사용되는것들이였습니다.

현재 문제, 즉 허용하지도 않았는데 어떻게 .http 는 가져왔는가에는 적합하지 않았죠.

### CURL방식인가?

하지만 이때 문득 떠오른게 있었습니다.  CURL 이라는게 있다는것을요.

해당 요청을 떠올리게된 계기는 젠킨스 파이프라인을 작성할때 여러번 사용했었고, 해당 출력에서 여러가지 정보값들이 나왔던것들을 떠올렸기에 해당 방식이 수상하다고 여겨졌습니다.

그래서 한번 CURL 요청을 시도해봤습니다.

<MDXImage height="232" width="694" alt="undefined" src="cc88f94f-ee28-43ba-a23c-5396e0dc00a2.png" type="main" />

그리고 역시나 예상에 맞게 CURL 요청을 보낼시에 모든 정보값들이 그대로 나왔습니다.

### 어째서 CURL은 모든 정보값을 가져올수있는가?

그렇다면 CURL은 어떻게 CORS 정책을 무시하고 모든 정보값을 가져올수 있었을까요?

찾아보면 찾아볼수록 제가 아직 뉴비라는걸 깨닫기만 했습니다.

[해당 게시글에서](https://inpa.tistory.com/entry/WEB-%F0%9F%93%9A-CORS-%F0%9F%92%AF-%EC%A0%95%EB%A6%AC-%ED%95%B4%EA%B2%B0-%EB%B0%A9%EB%B2%95-%F0%9F%91%8F) 기본적인 정보를 얻을 수 있습니다.

하지만 저는 여러모로 삽질해서 알아냈습니다… 😭

1. CORS는 브라우저 및 자바스크립트에서 적용된다.
   즉 보안을 적용시키게 되면은 CORS 정책에 대해서 알아야한다.
2. CURL은 커맨드라인 도구이므로 단순히 HTTP 요청을 보내는데 사용된다.
   즉 CORS 정책을 따르지 않는다.
3. 그러므로 CURL는 보안성 정책을 따르지 않기 때문에 모든 정보값들을 가져올 수 있다.

위와 같은 3가지 이유로 .http는 모든 정보값을 가져올 수 있었던겁니다.
